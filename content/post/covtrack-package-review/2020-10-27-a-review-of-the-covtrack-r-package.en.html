covtrack is an R-package developed by Ketan Kabu to provide an overview of the COVID-19 cases and deaths in USA, Brazil, India and Australia through a Shiny application embedded within the package.

While the application is primarily aimed at viewing the status of COVID-19 cases in the countries mentioned above, he also hopes that it will also be used to influence decision making in the future.

Installing the package was a breeze, with all the required dependencies also being installed along with the package ‚≠ê The README file on the package Git Repository gave clear instructions on how to install the package. However, I was a bit confused about what I could do with the package and what functions I could use, since I couldn‚Äôt find this information on the README file üòï I also couldn‚Äôt find a publicly accessible website with the documentation and vignettes.

So I decided to directly launch the app exported by the covtrack package usng the launch_app() function, and received an error stating that a few functions used within the app couldn‚Äôt be found ü§î

Checking out the code ü§ìüíª

With the power of open source, I cloned the Git repository of the package and did some snooping around to try and launch the Shiny app and see what interesting insights it had to offer.

Turns out that I only had to attach the covtrack package within itself, to use the functions eported by the package in the embedded app. A simple fix for a simple mistake üîß.

Although I fixed this error, I still couldnt get the plots within the app to appear. With limited time and resources I decided that this is a fix for a another day.

What I loved about the app ‚ù§Ô∏è

The app provided all the information expected of it, such as a comparison of the trend of COVID-19 cases and deaths due to COVID-19 in the selected countries, as well as a comparison of death rates per confirmed cases.

But what I loved most about the app is :

The structure of the app
The app was structured in a very user-friendly format with each menu item having a user guide explaining the visualization and instructions to use.

The interactive elements in the app
The user was allowed to select the countries to be included in the plots and also select the date ranges to filter the plot.

What can be improved ‚ùì

The area that I feel needs most improvement is the documenation. It was such a shame that all the documentation and vignettes were included within the app but not exposed to the public.
Although this is a fantastic app, the lack of documentation stating what functions are available to use and examples of usage lets it down.

Another area that could be improved is by using functions within the Shiny application embedded within the package to minimize repetitive code.

The data reading and wrangling within the application could also be done external to the package and the cleaned data files included within the package to minimize the size of the package and processing and loading time.

More test cases can also be included to test functionality, possible input arguments and edge cases for the exported functions.

Conclusion ü§ù

The covtrack R-package is definitely and interesting package aimed at comparing COVID-19 cases across USA, Brazil, India and Australia, which can then be used to compare the similarities and differences in trends based on the different approaches followed by the countries to manage the spread. While there are still a few areas for improvement, the package is definitely a great idea in the making üëè

Reviewing the package was also a great learning experience for me in terms of debugging an application, identifying and resolving errors as well as understanding a completely different coding-style to mine.

Hope you enjoyed my review!
